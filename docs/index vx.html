Coupler — How It Works

Coupler analyzes TrueType and OpenType fonts and computes optical kerning corrections based on the actual glyph contour geometry. No manual pair-by-pair editing — all values are derived mathematically from the outline shapes.

Workflow

  1. Drop a TTF or OTF file onto the drop zone (or click to browse).
  2. Adjust parameters if needed, then click Compute Kerning.
  3. Inspect results in the Preview and the Coupling Table.
  4. Export as CSV for use in font editors or layout systems.

How Kerning Is Computed

Each glyph's bounding silhouette is measured at N horizontal zones using Bézier intersection. A baseline reference distance is derived from the pair o+o (lowercase) or O+O (uppercase). The correction for every pair is: correction = base − pairMean, where pairMean is the average gap across all valid zones of the pair.

Negative correction = move closer. Positive = move apart.

Parameters

Zones (subtlety)
Number of horizontal slices across the em height. More zones = finer contour detail captured, but slower computation.
Smooth (diagonals and counter)
Reciprocal smoothing scale: 0 = off, 99 = maximum smoothing. Controls the maximum step between adjacent zone margins. Higher values produce smoother contour profiles for diagonal strokes and open counters.
Min gap (no touch)
Prevents any correction from bringing glyphs closer than this percentage of the UPM. Acts as a collision guard.
Blur (less details)
Each zone is sampled at N sub-zones internally and the results averaged. Reduces edge quantization artifacts where curves cross zone boundaries obliquely.
Round module (cadence)
Snaps correction values to multiples of N. Useful for systems that prefer a regular grid of kerning values (e.g. multiples of 46 units).
Threshold
Corrections with absolute value below this threshold are discarded (set to 0). Reduces noise in the kerning table.

Function Reference

Loading a Font

Drop a TTF or OTF file onto the drop zone, or click to browse. The file is read as an ArrayBuffer, parsed by opentype.js, and a Base64 data URL is stored for later use in the test page. The kerning analysis runs automatically after a successful load.

Computing Kerning — Four-Step Pipeline

  1. Measure margins — Every glyph is sliced into Zones equal horizontal bands. For each band, Bézier intersection geometry (cubic and quadratic) determines the leftmost and rightmost ink extent. Each zone margin is the average of Blur sub-zones, which smooths staircase artifacts where curves cross band boundaries at oblique angles.
  2. Smooth & shape — A reciprocal step-limit pass (Smooth) prevents sudden jumps between adjacent zone margins, anchoring from the zone of maximum glyph extent outward. An optional quadratic Bowl then bends the entire profile: positive values bow margins outward (convex); negative values bow them inward (concave).
  3. Establish baselines — The self-pair of the reference glyph (default o+o for LC, O+O for UC) is measured and its mean gap becomes the target spacing. A global Tracking offset shifts the effective baseline before corrections are computed.
  4. Compute pair corrections — For every pair, the mean zone gap (pair mean) is computed. Correction = baseline − pair mean. Corrections that would violate the Min gap floor are capped (⚑). Corrections below the Threshold are zeroed out. The final value is rounded to the nearest multiple of the Round module.

Preview Canvas

The preview renders the entered text directly onto an HTML Canvas using opentype.js glyph paths. Kerning corrections are applied as advance-width offsets per character pair. Optional overlays show the computed left (amber) and right (green) margin bands per zone, the typographic baseline, and the x-height line. In Pair mode (⊙ Pair), the selected table row is shown in a context string — navigate rows with the arrow keys, exit with Escape.

Spacing Corrections Tab

For each glyph, the self-pair correction (A+A, n+n, …) is extracted from the kerning table. Two adjustment suggestions are derived: ΔAdvanceWidth (half the self-kern, split symmetrically between both sides) and ΔSidebearing (a quarter of the self-kern, shifting the glyph within its bounding box). If applied to the font, the self-pair correction approaches zero.

Export

Downloads all kerning pairs as a semicolon-delimited CSV with a parameter header. Each line contains left glyph, right glyph, correction value, class (LC / UC / mixed), and zone breakdown. Capped pairs are flagged with [cap].

Glossary

UPM
Units Per Em — the internal coordinate unit of a font. All distances (margins, corrections, gap floors) are expressed in UPM units.
Glyph
A single character as a vector path. Each glyph carries an advance width and optionally one or more Unicode code points.
Advance Width
The horizontal distance the cursor moves after placing a glyph, including any side-bearings.
Kerning
A pair-specific adjustment to the space between two glyphs. Negative = tighter; positive = looser.
Zone
One equal-height horizontal slice across the full em range (ascender to descender). More zones capture finer vertical contour detail at the cost of computation time.
Margin
The gap between the outermost ink edge of a glyph and its advance-width boundary — measured separately on the left and right side for every zone.
Pair Mean
The arithmetic mean of all valid zone gaps for a pair: right margin of glyph A + left margin of glyph B, averaged across every zone where both glyphs have ink.
Base Value
The reference pair mean from the self-pair of the base glyph (default o for LC, O for UC). It is the target spacing against which all corrections are computed.
LC / UC
Lowercase / Uppercase (including lining figures). Determines which base value is used for a given pair.
Mixed
A pair classification where one glyph is LC and the other is UC; the LC base value is used.
Bézier Curve
The parametric curve type used in OpenType outlines. Quadratic curves appear in TrueType fonts; cubic curves appear in CFF/OTF fonts. Coupler evaluates both types geometrically to extract ink extents.
Counter
The enclosed interior void of a glyph (e.g., the hole in "o" or "p"). Measured via an off-screen canvas pixel scan.
Print Area
The total ink area of a glyph (sum of ink-covered pixels in the zone pixel map), expressed in font-unit² equivalents.
Smooth
Reciprocal smoothing strength (0 = off, 99 = maximum). Limits the maximum step allowed between adjacent zone margins, anchoring from the zone of maximum glyph extent and spreading outward.
Bowl
Quadratic curvature applied to the smoothed margin profile after smoothing. Positive: margins bow outward (larger away from the anchor zone). Negative: margins bow inward (smaller away from the anchor zone).
Min Gap %
The minimum allowed zone gap after applying a kerning correction, expressed as a percentage of UPM. Acts as a collision guard; pairs that hit this floor are marked with ⚑.
Blur
Number of sub-zones computed per zone. Zone margins are the average of all sub-zone readings, suppressing quantization artifacts where curves cross zone boundaries obliquely.
Round Module
Kerning values are rounded to the nearest multiple of this number (e.g., 46 → values like −92, −46, 0, 46, 92). Use 1 for plain integer rounding.
Threshold
Minimum absolute value for a correction to be retained. Corrections in the range (−threshold … +threshold) are discarded as noise.
Tracking
A global offset added to the effective base value before computing corrections. Negative tightens all spacing; positive loosens it.
Self-Kern
The kerning correction of a glyph paired with itself (A+A). A non-zero self-kern indicates unbalanced sidebearings in the original font.
ΔAdvanceWidth
Suggested advance-width adjustment to balance the self-pair: half the self-kern correction, applied symmetrically to both sides.
ΔSidebearing
Suggested sidebearing shift (glyph position within its bounding box): one quarter of the self-kern correction.
Small Caps (SC)
Small-capital glyph variants, identified by names ending in .sc or .smcp.
OSF
Oldstyle Figures — non-lining numerals, identified by names ending in .osf or PUA code points U+F730–U+F739.
Pair Limit
Maximum number of kerning pairs to compute, prioritized by frequency of occurrence in real-world text (source: Andre Fuchs kerning-pairs corpus). Set to 0 to compute all pairs.
Capped (⚑)
The computed correction was reduced because it would have violated the Min Gap floor. The tightest-zone gap minus the Min Gap determines the maximum allowable negative correction.
Pair Preview Mode
Clicking a table row (or pressing ⊙ Pair) activates pair preview: the selected pair is shown in the context string oABo nABn OABO HABHAB. Navigate with ↑↓ arrow keys; exit with Escape.
