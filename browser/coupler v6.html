Coupler — How It Works

Coupler analyzes TrueType and OpenType fonts and computes optical kerning corrections based on the actual glyph contour geometry. No manual pair-by-pair editing — all values are derived mathematically from the outline shapes.

Workflow

  1. Drop a TTF or OTF file onto the drop zone (or click to browse).
  2. Adjust parameters if needed, then click Compute Kerning.
  3. Inspect results in the Preview and the Coupling Table.
  4. Export as CSV for use in font editors or layout systems.

How Kerning Is Computed

Each glyph's bounding silhouette is measured at N horizontal zones using Bézier intersection. A baseline reference distance is derived from the pair o+o (lowercase) or O+O (uppercase). The correction for every pair is: correction = base − pairMean, where pairMean is the average gap across all valid zones of the pair.

Negative correction = move closer. Positive = move apart.

Parameters

Zones (subtlety)
Number of horizontal slices across the em height. More zones = finer contour detail captured, but slower computation.
Smooth (diagonals and counter)
Reciprocal smoothing scale: 0 = off, 99 = maximum smoothing. Controls the maximum step between adjacent zone margins. Higher values produce smoother contour profiles for diagonal strokes and open counters.
Min gap (no touch)
Prevents any correction from bringing glyphs closer than this percentage of the UPM. Acts as a collision guard.
Blur (less details)
Each zone is sampled at N sub-zones internally and the results averaged. Reduces edge quantization artifacts where curves cross zone boundaries obliquely.
Round module (cadence)
Snaps correction values to multiples of N. Useful for systems that prefer a regular grid of kerning values (e.g. multiples of 46 units).
Threshold
Corrections with absolute value below this threshold are discarded (set to 0). Reduces noise in the kerning table.

Preview Toggles

⟷ Kerning ON/OFF
Switch between Coupler kerning and no kerning to compare spacing visually.
Margins
Overlay the computed left (amber) and right (green) margin zones on each glyph.
Baseline
Show the typographic baseline as a dashed amber line.
x-Height
Show the x-height line in blue (if the font provides this metric).

Table Filter

Exact
Case-sensitive exact glyph name match. "o" finds only the pair where left or right is exactly "o".
A B (two terms)
Shows only the pair where left=A and right=B exactly.
.* Regex
Full JavaScript regular expression against glyph names.
